+# Summary of Changes
+
+## Problem
+The existing code in `/testbed/packages/ui/src/utils/ThemeUtil.ts` was always including a Google Fonts import for the Inter font, even when a custom font family was provided via the `--w3m-font-family` CSS variable. This resulted in unnecessary network requests to third-party services.
+
+## Solution
+Modified the `createRootStyles` function to conditionally include the Google Fonts import:
+
+### Key Changes
+1. **Added logic** to detect when a custom font family is provided via `themeVariables?.['--w3m-font-family']`
+2. **Conditional import**: The Google Fonts CDN import is only included when no custom font is specified
+3. **Edge case handling**: Empty strings and whitespace-only values are treated as "no custom font"
+
+### Implementation Details
+```typescript
+function createRootStyles(themeVariables?: ThemeVariables) {
+  const customFontFamily = themeVariables?.['--w3m-font-family'];
+  // Only include the Google Fonts import if no custom font family is provided
+  const hasCustomFont = customFontFamily && customFontFamily.trim();
+  const fontImport = hasCustomFont ? '' : `@import url('https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&display=swap');\n`;
+  
+  return {
+    core: css`${unsafeCSS(fontImport)}
+      // ... rest of the CSS content
+    `
+  };
+}
+```
+
+## Benefits
+- ✅ **Reduces network requests**: No unnecessary Google Fonts CDN calls when custom fonts are used
+- ✅ **Preserves default behavior**: Inter font is still loaded when no custom font is specified
+- ✅ **Backward compatible**: Existing users without custom fonts are unaffected
+- ✅ **Optimized performance**: Minimal runtime overhead for the conditional check
+
