pub fn header(input: ParserInput) -> Option<String> {
    strip_nom(map_res(
        delimited(
            tag(&b"%PDF-"[..]),
            take_while(|c: u8| !b"\r\n".contains(&c)),

The issue is:
1. It takes everything after `%PDF-` until `\r` or `\n`
2. Then tries to convert to UTF-8 string via `str::from_utf8`
3. If there are binary bytes (like `\xb0\x9f\x92\x9c...`), `str::from_utf8` fails, and the whole `map_res` fails → `header` returns `None` → `InvalidFileHeader` error

The fix should be to only parse the version number from the header line (like `1.3`), ignoring trailing binary bytes. Looking at the PR description:
- pypdf: Only validates the 5-byte `%PDF-` prefix
- pdf.js: Reads version chars until whitespace (byte ≤ 0x20) or 7 chars max
- qpdf: Uses regex `[0-9]+\.[0-9]+` to extract version digits

The approach should be: after `%PDF-`, extract the version number (digits and dot) until we hit a whitespace character or other non-version character, then skip the rest of the line.
